Brain and Brawn - Level Making Guide

Basics:

To create levels, go to test.html, click past the title screen, and hit "Create Levels"

Although you can create levels on your phone, it is recommended that you use a computer browser if you want to load/save level files.

Loading and Saving

Brain and Brawn levels are JSON files. To prevent bugs, it is recommended that you never edit the JSON files directly, and instead edit by loading into the editor and saving over your previous version. Note that the level editor doesn't let you set star requirements, but if you're bold you can do this inside the JSON (or just let me know).

To load a level: Click the "choose a file" button above the game canvas. Select the JSON level file you want to load. If you load something other than a BnB level, the game will crash.

To save a level: Click the SAVE disk button to open a new browser tab with a "Download" button. Press Download to save your level as a JSON file.

Tools and Brushes:

Special Objects:

  • Brush Window (top left): Displays the currently selected brush.
  • Play Button: Tap to play your WIP level. The game will crash if you have 2+ Brainy or 2+ Brawny.
  • Eraser: Tap to select the eraser (Brush window will become empty). Tap on a tile to erase the sprite.
  • SAVE: See loading/saving section above.
  • Flip Diagonally: Flip your level diagonally.
  • Flip Vetically: Flip your level vertically.
  • **(NOTE: you can perform a horizontal flip with the following flip[ sequence: diagonal,vertical,digonal)**
  • Adjust Square: Enter the "Adjust Menu"

Brushes:

To use a brush, tap on it to select it (note the Brush Window will change), then tap on the tile where you'd like to place that object.

Adjust Menu:

  • Left/Right: Adjust level width.
  • Up/Down: Adjust level height.
  • Boundary Death: TRUE by default. Set to FALSE to prevent Brainy/Brawny from flying out into space. This is useful if you want to quickly test/iterate on a level idea without creating walls around the border, but keep in mind that in the actual game this is always set to true.
